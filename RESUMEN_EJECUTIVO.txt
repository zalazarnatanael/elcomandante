🔄 FLUJO NOTION → GITHUB → DESARROLLO (4 STAGES)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  STAGE 1: NOTION → GitHub Issue
  ├─ Script: auto_expand_ready_ferreteria.js
  ├─ Trigger: Cron o manual
  └─ Resultado: Issue + label "from-notion"

  STAGE 2: PLAN (Análisis READ-ONLY)
  ├─ Trigger: Label "from-notion"
  ├─ Modelo: opencode/trinity-large-preview-free (default)
  ├─ Output: Plan comentado en GitHub
  ├─ Contexto: Feedback histórico
  └─ Label resultado: "awaiting-human-intervention"

  STAGE 2B: FEEDBACK LOOP (Iterativo)
  ├─ Trigger: Nuevo comentario (no bot)
  ├─ Acción: Re-analiza con feedback
  └─ Vuelve a STAGE 2

  STAGE 3: BUILD (Implementación WRITE)
  ├─ Trigger: Label "ready-for-development"
  ├─ Modelo: github-copilot/claude-haiku-4.5 (default)
  ├─ Worktree: ~/openclaw-workspace/worktrees/issue-X
  ├─ Cambios: Git aislado
  └─ Resultado: PR creado + label "pr-generated"

  STAGE 4: MERGE → COMPLETION
  ├─ Trigger: PR merged
  ├─ Acción: Actualiza Notion (Estado = "Completada")
  └─ FIN: Tarea completada

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🤖 MODELOS IA Y FALLBACKS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  PLAN MODE:
  ├─ Primary: opencode/trinity-large-preview-free
  ├─ Fallback 1: opencode/minimax-m2.5-free
  ├─ Fallback 2: opencode/big-pickle
  └─ Límite: Si todos fallan → FAIL

  BUILD MODE:
  ├─ Primary: github-copilot/claude-haiku-4.5
  ├─ Fallback 1: opencode/trinity-large-preview-free
  ├─ Fallback 2: opencode/minimax-m2.5-free
  └─ Límite: Si todos fallan → FAIL

  TRIGGERS DE FALLBACK:
  ├─ "model not found"
  ├─ "providermodelnotfounderror"
  ├─ "request too large"
  ├─ "tpm" (Token Per Minute)
  └─ "provider not found"

  CONSTRAINT ESPECIAL:
  ├─ PLAN mode es READ-ONLY
  ├─ Detecta intentos de write
  ├─ Si se detectan: Fuerza fallback
  └─ Patrones: ← Edit, file diffs, etc.

🔗 INTEGRACIONES EXTERNAS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  GitHub API:
  ├─ Rate limit: 5000 req/hora (authenticated)
  ├─ Retry: 4 intentos con exponential backoff
  ├─ Endpoints: issues, PRs, comments, webhooks
  └─ Error handling: withRetry() + clasificación

  Notion API:
  ├─ Database query (búsqueda tareas)
  ├─ Pages retrieve/update (estado)
  ├─ Sin rate limiting visible
  └─ Mapeo: notion_page_id en issue body

  OpenCode (CLI):
  ├─ Invocación: spawn("opencode run ...")
  ├─ Comunicación: stdin/stdout/stderr
  ├─ Session: Persistida en ~/.opencode/sessions/
  └─ Modelos: Configurable via --model

  Telegram:
  ├─ Notificaciones de error
  ├─ Resumen de cambios
  ├─ Búsqueda: env vars + config JSON
  └─ HTTPS POST: api.telegram.org/bot{token}/sendMessage
